# Security Policy

## Supported version

Only the latest `main` branch is supported during the MVP.

## Reporting

Do not open public issues containing API keys, private textbooks, student data, or vulnerability details. Report sensitive problems privately to the repository owner.

## Operational requirements

- Never commit `.env` or API keys.
- Do not use copyrighted textbooks without written authorization.
- Do not expose the pilot directly to the internet without HTTPS, authentication, and rate limiting.
- Treat generated educational content as draft material until reviewed by an instructor.
